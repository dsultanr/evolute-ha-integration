# Интеграция Evolute для Home Assistant

[![hacs_badge](https://img.shields.io/badge/HACS-Custom-41BDF5.svg)](https://github.com/hacs/integration)

Интеграция Home Assistant для автомобилей Evolute (i-Joy, i-Pro, i-Sky, i-Space и т.д.) через облачный сервис app.evassist.ru.

<img width="1073" height="1653" alt="image" src="https://github.com/user-attachments/assets/bacb9294-98a1-44e8-bc8f-9967ff0c5f67" />

## Возможности

- **30+ сенсоров**, включая:
  - Заряд тяговой батареи и запас хода
  - Напряжение 12В АКБ, температуры (батарея, охлаждающая жидкость, салон, климат, снаружи)
  - Одометр, скорость, обороты вентилятора климат-контроля
  - GPS-локация (широта, долгота, высота, скорость, спутники, HDOP, курс)
- **Бинарные сенсоры** для дверей, багажника, фар, зажигания, центрального замка, климата, обогрева лобового стекла, статуса "онлайн" и движения
- **Кнопки управления**: центральный замок, обогрев, охлаждение, багажник, звуковой сигнал/поиск, старт/отмена подготовки к поездке
- **Трекер местоположения** на карте Home Assistant
- Поддержка нескольких автомобилей на одном аккаунте

## Авторизация: почему не логин/пароль

Официальное приложение Evolute входит по номеру телефона и SMS-коду, но перед отправкой кода сайт требует пройти Yandex SmartCaptcha. Автоматизировать это программно ненадёжно (или является обходом защиты, чего эта интеграция сознательно не делает).

Вместо этого используется тот же подход, что и в проекте [evolute_proxy](https://github.com/hackruu/evolute_proxy) и Android-приложении [i-joy](https://github.com/wzd2002/i-joy): вы **один раз** входите в app.evassist.ru через браузер и копируете два cookie-токена. Дальше интеграция сама обновляет `access_token` через `refresh_token` (он живёт около 90 дней) и сохраняет обновлённые значения, так что повторно вставлять токены обычно не требуется — разве что `refresh_token` будет отозван сервером.

### Как получить токены

1. Откройте https://app.evassist.ru в браузере и войдите в свой аккаунт (телефон + SMS-код)
2. Откройте инструменты разработчика (F12) → вкладка **Application** (Chrome) или **Storage** (Firefox)
3. Слева выберите **Cookies** → `https://app.evassist.ru`
4. Найдите и скопируйте значения двух cookies:
   - `evy-platform-access`
   - `evy-platform-refresh`
5. Вставьте их в форму настройки интеграции в Home Assistant

## Установка

### HACS (Рекомендуется)

1. Откройте HACS в вашем Home Assistant
2. Перейдите в раздел "Интеграции"
3. Нажмите три точки в правом верхнем углу
4. Выберите "Пользовательские репозитории"
5. Добавьте URL репозитория: `https://github.com/dsultanr/evolute-ha-integration`
6. Категория: "Интеграция"
7. Нажмите "Добавить"
8. Найдите "Evolute"
9. Нажмите "Загрузить"
10. Перезапустите Home Assistant

### Ручная установка

1. Скопируйте директорию `custom_components/evolute` в директорию `custom_components` вашего Home Assistant
2. Перезапустите Home Assistant

## Настройка

1. Перейдите в **Настройки** → **Устройства и службы**
2. Нажмите **"+ ДОБАВИТЬ ИНТЕГРАЦИЮ"**
3. Найдите **"Evolute"**
4. Вставьте **Access Token** и **Refresh Token** (см. раздел выше)
5. Нажмите **Отправить**

Все автомобили вашего аккаунта появятся автоматически со всеми доступными сенсорами.

## Сенсоры

### Обычные сенсоры

**Батарея и запас хода:**
- Заряд тяговой батареи (%)
- Запас хода (км)
- Уровень топлива (%) и запас хода по топливу (км) — поле присутствует в API даже для электромобилей
- Напряжение 12В АКБ

**Температуры:**
- Температура батареи
- Температура охлаждающей жидкости
- Температура в салоне
- Температура снаружи
- Текущая и целевая температура климат-контроля
- Скорость вентилятора климат-контроля

**Движение и GPS:**
- Одометр
- Скорость
- Широта, долгота, высота
- Количество спутников, HDOP, курс

**Прочее:**
- Время последнего выхода на связь

### Бинарные сенсоры

- Зажигание
- Центральный замок
- Двери (водительская, пассажирская, задние левая/правая)
- Багажник
- Фары
- Климат-контроль включён
- Обогрев лобового стекла
- Онлайн (связь с автомобилем)
- Движение (автомобиль не на стоянке)
- Идёт подготовка к поездке (прогрев/охлаждение по таймеру)

## Кнопки управления

- **Переключить замок** — центральный замок (открыть/закрыть)
- **Переключить обогрев** — включить/выключить обогрев салона
- **Переключить охлаждение** — включить/выключить кондиционер
- **Переключить багажник** — открыть/закрыть багажник
- **Сигнал и поиск** — короткий гудок и мигание аварийкой
- **Старт подготовки к поездке** — запустить сценарий прогрева/охлаждения по таймеру
- **Отмена подготовки к поездке** — не подтверждено в перехваченном трафике, добавлено по аналогии; если не сработает — сообщите в Issues

Важно: у API Evolute нет отдельных команд "включить"/"выключить" для замка, обогрева, охлаждения и багажника — есть только команда-переключатель (toggle). Кнопки просто вызывают её, поэтому текущее состояние стоит проверять по соответствующему бинарному сенсору перед нажатием.

## Визуализация автомобиля

Интеграция включает готовую **схему автомобиля сверху** (переиспользована из [prizrak-ha-integration](https://github.com/dsultanr/prizrak-ha-integration) — расположение дверей совпадает):

- SVG-файлы автоматически копируются в `/config/www/evolute/` при запуске интеграции
- 🔵 **Синий контур**: центральный замок закрыт
- 🟡 **Жёлтая/оранжевая подсветка**: конкретная дверь или багажник открыты
- 🔧 **Иконка двигателя**: зажигание включено
- ❄️ **Иконка климата**: обогрев/охлаждение активны

Капот на схеме не подсвечивается — API Evolute не отдаёт его статус.

### Пример карточки Dashboard

Готовый пример: [examples/car-card.yaml](examples/car-card.yaml). Замените `CAR_ID_HERE` на ID вашего автомобиля (виден в entity_id любого сенсора, например `sensor.evolute_68f2.._battery_percentage`).

В отличие от prizrak, кнопки управления в примере — обычные `type: button` карточки, без зависимости от HACS `card-mod`/`button-card`: у Evolute нет отдельных команд "включить"/"выключить", только toggle, поэтому сложная логика с шаблонизацией entity_id не нужна.

## Примеры автоматизаций

### Уведомление о низком заряде батареи

```yaml
automation:
  - alias: "Evolute: Низкий заряд батареи"
    trigger:
      - platform: numeric_state
        entity_id: sensor.evolute_CAR_ID_HERE_battery_percentage
        below: 20
    action:
      - service: notify.mobile_app
        data:
          title: "Низкий заряд автомобиля"
          message: "Заряд батареи Evolute: {{ states('sensor.evolute_CAR_ID_HERE_battery_percentage') }}%"
```

### Уведомление об открытой двери

```yaml
automation:
  - alias: "Evolute: Открыта водительская дверь"
    trigger:
      - platform: state
        entity_id: binary_sensor.evolute_CAR_ID_HERE_door_fl
        to: "on"
    action:
      - service: notify.mobile_app
        data:
          title: "Оповещение о машине"
          message: "Водительская дверь была открыта!"
```

### Автоматический прогрев по расписанию

```yaml
automation:
  - alias: "Evolute: Прогрев перед работой"
    trigger:
      - platform: time
        at: "07:30:00"
    condition:
      - condition: state
        entity_id: binary_sensor.evolute_CAR_ID_HERE_online
        state: "on"
    action:
      - service: button.press
        target:
          entity_id: button.evolute_CAR_ID_HERE_prepare_start
```

## Устранение неполадок

### Интеграция просит повторную авторизацию (reauth)

Это означает, что `refresh_token` был отозван сервером (например, после смены пароля или выхода из всех сессий). Повторите шаги получения токенов из браузера и введите новые значения в открывшейся форме — переустанавливать интеграцию не нужно.

### Интеграция не загружается

1. Проверьте логи Home Assistant: **Настройки** → **Система** → **Логи**
2. Ищите ошибки, связанные с "evolute"
3. Убедитесь, что скопированные токены не содержат лишних пробелов

### Сенсоры показывают "недоступен"

1. Проверьте интернет-соединение
2. Убедитесь, что app.evassist.ru доступен и автомобиль виден в приложении
3. Проверьте логи Home Assistant на наличие ошибок HTTP-запросов
